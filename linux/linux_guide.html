Introdução ao Linux

Conceitos fundamentais, estrutura do SO e o terminal

O que é Linux?

Linux é um kernel de sistema operacional open-source criado por Linus Torvalds em 1991. Distribuições Linux (distros) empacotam o kernel com ferramentas GNU e interfaces para formar um SO completo.

  • Kernel: núcleo que gerencia hardware, memória e processos.
  • Shell: interpretador de comandos (bash, zsh, sh).
  • Distros populares: Ubuntu, Debian, Fedora, Arch, CentOS/RHEL, openSUSE.
  • Multiusuário e multitarefa: vários usuários e processos simultâneos.
  • Tudo é arquivo: dispositivos, sockets e processos são representados como arquivos.

Estrutura de Diretórios (FHS)

DiretórioDescrição

/binBinários essenciais (ls, cp, mv…)

Permissões de Arquivo

chmod, chown, chgrp e notação octal e simbólica

Entendendo Permissões

-rwxr-xr-- 1 franklin devs 4096 Jan 1 arquivo.sh │││││││││││ │└┤└┤└──┤ └── outros (r--) │ │ └──┤ └─── grupo (r-x) │ └────┤ └──── dono (rwx) └──────┘ └───── tipo: - arquivo, d diretório, l link
  • r (read) = 4: leitura
  • w (write) = 2: escrita
  • x (execute) = 1: execução
  • Três grupos: user, group, other (e all)

chmod — Alterar Permissões

# Notação simbólica $ chmod u+x script.sh # dono: adiciona execução $ chmod go-w arquivo # grupo+outros: remove escrita $ chmod a+r arquivo # todos: adiciona leitura # Notação octal $ chmod 755 script.sh # rwxr-xr-x $ chmod 644 config.txt # rw-r--r-- $ chmod 700 privado.sh # rwx------ $ chmod 600 chave.pem # rw-------

chown e chgrp

# Mudar dono $ chown franklin arquivo.txt $ chown -R franklin:devs projeto/ # dono:grupo recursivo # Mudar apenas o grupo $ chgrp devs arquivo.txt # Permissões especiais $ chmod 4755 prog # SUID: executa como dono $ chmod 2755 dir/ # SGID: herda grupo $ chmod 1777 /tmp # Sticky bit: só dono deleta

Redirecionamento & Pipes

stdin/stdout/stderr, pipes, grep, sed, awk e xargs

Redirecionamento de E/S

# Redirecionar saída (sobrescreve) $ ls -la > lista.txt # Redirecionar saída (acrescenta) $ echo "nova linha" >> log.txt # Redirecionar entrada $ sort < dados.txt # Redirecionar stderr $ comando 2> erros.txt # Redirecionar stdout e stderr $ comando &> tudo.txt $ comando 2>&1 | less # Descartar saída $ comando > /dev/null 2>&1

Pipes e Filtros Essenciais

# Pipe: saída de um → entrada do próximo $ ls -la | grep ".sh" # filtra arquivos .sh $ ps aux | grep nginx # procura processo nginx $ cat /etc/passwd | sort # ordena linhas # grep — busca por padrão $ grep -i "erro" log.txt # case insensitive $ grep -r "TODO" src/ # recursivo $ grep -n "fail" sys.log # mostra nº de linha $ grep -v "debug" log # inverte (exclui match) $ grep -E "err|warn" log # expressão regular

sed, awk e xargs

# sed — stream editor $ sed 's/antigo/novo/g' arquivo $ sed -i 's/localhost/127.0.0.1/g' config $ sed -n '5,10p' arquivo # imprime linhas 5-10 # awk — processamento de campos $ awk '{print $1}' /etc/passwd # 1ª coluna $ awk -F: '{print $1,$3}' /etc/passwd # xargs — passa args para outro cmd $ find . -name "*.tmp" | xargs rm

Systemd & Serviços

systemctl, journalctl, unit files e gerenciamento de serviços

systemctl — Controle de Serviços

$ systemctl start nginx $ systemctl stop nginx $ systemctl restart nginx $ systemctl reload nginx # recarrega config sem parar $ systemctl status nginx # status detalhado $ systemctl enable nginx # ativa no boot $ systemctl disable nginx # desativa no boot $ systemctl is-active nginx $ systemctl list-units --type=service $ systemctl daemon-reload # relê unit files

Criando um Unit File Personalizado

# /etc/systemd/system/meuapp.service [Unit] Description=Meu Aplicativo Web After=network.target [Service] Type=simple User=www-data WorkingDirectory=/opt/meuapp ExecStart=/usr/bin/python3 app.py Restart=on-failure RestartSec=5 [Install] WantedBy=multi-user.target

journalctl — Logs do Sistema

$ journalctl -u nginx # logs do nginx $ journalctl -f # segue logs em tempo real $ journalctl --since "1 hour ago" $ journalctl -p err # apenas erros $ journalctl -b # desde o último boot $ journalctl --no-pager -u ssh

UFW — Uncomplicated Firewall

$ ufw enable # ativa o firewall $ ufw status verbose $ ufw allow 22/tcp # permite SSH $ ufw allow 80,443/tcp $ ufw deny 23/tcp # bloqueia Telnet $ ufw allow from 192.168.1.0/24 # permite sub-rede $ ufw delete allow 80/tcp $ ufw default deny incoming # política padrão $ ufw default allow outgoing

Hardening SSH e fail2ban

# /etc/ssh/sshd_config — configurações de segurança PermitRootLogin no PasswordAuthentication no PubkeyAuthentication yes Port 2222 # porta não padrão MaxAuthTries 3 # fail2ban — bloqueia IPs após tentativas falhas $ systemctl enable --now fail2ban $ fail2ban-client status sshd # status do jail SSH $ fail2ban-client set sshd unbanip 1.2.3.4

Armazenamento & LVM

df, du, fdisk, mount, fstab e Logical Volume Manager

Disco e Sistema de Arquivos

# Uso de disco $ df -h # uso de partições $ du -sh /var/log # tamanho do diretório $ du -sh * | sort -rh # ordena por tamanho $ lsblk # lista dispositivos de bloco $ blkid # UUID e tipo de filesystem # Montar e desmontar $ mount /dev/sdb1 /mnt/dados $ umount /mnt/dados $ mount -a # monta tudo do /etc/fstab

LVM — Logical Volume Manager

# Camadas: PV → VG → LV # Physical Volume $ pvcreate /dev/sdb $ pvs # Volume Group $ vgcreate dados_vg /dev/sdb $ vgextend dados_vg /dev/sdc # adiciona disco # Logical Volume $ lvcreate -L 50G -n lv_dados dados_vg $ mkfs.ext4 /dev/dados_vg/lv_dados $ lvextend -L +20G /dev/dados_vg/lv_dados $ resize2fs /dev/dados_vg/lv_dados # redimensiona FS

/etc/fstab — Montagem Persistente

# UUID=xxx ponto_de_montagem fs_type opcoes dump pass UUID=abc123 /mnt/dados ext4 defaults 0 2 UUID=def456 /home ext4 defaults 0 2 # Obter UUID do disco $ blkid /dev/sdb1

Performance & Monitoramento

vmstat, iostat, sar, strace, tcpdump e análise de sistema

Ferramentas de Monitoramento

FerramentaUso
top / htopProcessos, CPU e memória em tempo real
vmstat 1Memória virtual, processos e I/O por segundo
iostat -xz 1Estatísticas de I/O de disco
free -hUso de memória RAM e swap
sar -u 1 5CPU histórica (sysstat)
strace -p PIDSyscalls de um processo em tempo real
lsof -p PIDArquivos abertos por um processo
tcpdump -i eth0Captura de pacotes de rede
dmesg | tailMensagens do kernel (hardware, erros)

Análise de Memória e CPU

# Memória $ free -h $ cat /proc/meminfo | grep -E "MemTotal|MemFree|Cached" # CPU $ nproc # número de cores $ lscpu # info detalhada da CPU $ uptime # load average (1, 5, 15 min) $ sar -u 1 10 # CPU a cada 1s por 10x # Discos $ iostat -xz 1 5 $ lsblk -o NAME,SIZE,TYPE,MOUNTPOINT

tcpdump — Captura de Pacotes

# Capturar na interface eth0 $ tcpdump -i eth0 # Filtrar por host $ tcpdump -i eth0 host 8.8.8.8 # Filtrar por porta $ tcpdump -i eth0 port 443 # Salvar captura $ tcpdump -i eth0 -w captura.pcap # Ler captura salva $ tcpdump -r captura.pcap
