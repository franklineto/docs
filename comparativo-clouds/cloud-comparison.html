Visão Geral 3 grandes provedores
Entenda o posicionamento, forças e o perfil de cada provedor antes de escolher seus serviços.
🟠
Amazon Web Services
AWS · aws.amazon.com
Pioneiro e líder absoluto de mercado (~32% de share). Maior portfólio de serviços (200+), ecossistema maduro e enorme comunidade. Escolha padrão para startups e empresas globais.
Maior portfólio Comunidade ampla Regiões globais Marketplace rico
🔵
Google Cloud Platform
GCP · cloud.google.com
Terceiro em market share (~12%), mas líder em IA/ML, Big Data e Kubernetes (criador do K8s). Infraestrutura de rede global do Google. Favorito de equipes de data science e engenharia de dados.
IA/ML líder BigQuery Rede Google Kubernetes nativo
🔷
Microsoft Azure
Azure · azure.microsoft.com
Segundo em market share (~23%), dominante em empresas com stack Microsoft (Windows, Office 365, Active Directory). Integração nativa com ferramentas corporativas e suporte a workloads híbridas.
Integração Microsoft Híbrido/OnPrem Compliance Enterprise Active Directory

A tabela abaixo compara os três provedores por categoria. Clique nas abas acima para explorar cada categoria com detalhes, vantagens, desvantagens e exemplos de uso hipotéticos.

Nota: os nomes dos serviços mudam entre provedores para a mesma função. Por exemplo: máquinas virtuais são EC2 na AWS, Compute Engine no GCP e Virtual Machines no Azure.

🟠 AWS EC2
Vantagens
  • Maior variedade de tipos de instância (750+ tipos)
  • Spot Instances com até 90% de desconto
  • Reserved Instances com planejamento de custos
  • Ecossistema maduro com Auto Scaling nativo
Desvantagens
  • Interface complexa para iniciantes
  • Preço pode ser alto sem planejamento
  • Cobrança de egresso de dados elevada
💡 Exemplo de uso

Uma fintech processa transações em instâncias c6i.xlarge com Auto Scaling Group que expande de 2 para 50 instâncias nos horários de pico, e usa Spot Instances para backtests noturnos a 1/10 do custo.

📄 Docs EC2 💰 Preços
🔵 GCP Compute Engine
Vantagens
  • Sustained Use Discounts automáticos (sem reservar)
  • Preemptible VMs mais baratas que Spots da AWS
  • Custom Machine Types (CPU/RAM exatos)
  • Rede privada do Google de alta performance
Desvantagens
  • Menos tipos de instância que AWS
  • Comunidade menor de suporte
  • Algumas regiões têm disponibilidade limitada
💡 Exemplo de uso

Uma empresa de renderização 3D usa instâncias com GPU T4 no GCP e aproveita os descontos automáticos por uso sustentado — qualquer VM que rodar mais de 25% do mês ganha desconto automático sem ação manual.

Uma empresa com 500 licenças Windows Server on-premises migra para Azure VMs usando Azure Hybrid Benefit, aproveitando as licenças já pagas e economizando até 40% no custo das VMs em relação a outras nuvens.

Auto Scaling & Load Balancing
Escalabilidade automática e distribuição de tráfego entre instâncias

🟠 AWS ASG + ALB
Vantagens
  • Auto Scaling Groups com políticas granulares

Vantagens
  • Global Load Balancer com IP anycast único
  • Managed Instance Groups com rolling updates
  • Scaling baseado em métricas customizadas do Cloud Monitoring
Desvantagens
  • Configuração do Cloud LB mais verbosa
  • Latência de scale-up ligeiramente maior
💡 Exemplo de uso

Um app global usa o Cloud Load Balancer com IP anycast: usuários no Brasil e no Japão acessam o mesmo IP, sendo roteados automaticamente para a região mais próxima sem configuração de DNS complexa.

📄 Cloud LB

Desvantagens
  • Tempo de provisionamento de novas instâncias maior
  • Custo do Application Gateway é elevado
💡 Exemplo de uso

Uma empresa de seguros usa VMSS com Application Gateway + WAF para proteger um portal web ASP.NET, escalando automaticamente de 3 para 30 instâncias em períodos de renovação anual de apólices.

S3 Cloud Storage Blob Storage
🟠AWSS3
Vantagens
  • Padrão de facto do mercado — amplamente compatível
  • S3 Intelligent-Tiering move dados automaticamente
  • Glacier para arquivamento a custo mínimo (~$1/TB/mês)
  • S3 Select para queries SQL direto nos objetos
Desvantagens
  • Egress (saída de dados) é cobrado e pode ser caro
  • Preço de requisições pode ser surpresa
💡 Exemplo de uso

Um portal de streaming de vídeo armazena 500TB de conteúdo no S3 Standard para vídeos recentes e move automaticamente para Glacier vídeos com mais de 2 anos, reduzindo custo de armazenamento em 80%.

📄 S3 Docs 💰 Preços
🔵GCPCloud Storage
Vantagens
  • Egress gratuito para outros serviços GCP
  • Classes de storage simples: Standard, Nearline, Coldline, Archive
  • Consistência forte por padrão (sem eventual consistency)
  • Integrado nativamente ao BigQuery e Dataflow
Desvantagens
  • Egress para fora do GCP também é cobrado
  • Menor ecossistema de ferramentas terceiras
💡 Exemplo de uso

Uma equipe de data engineering carrega 10TB de logs diários no Cloud Storage (Nearline) e processa diretamente no BigQuery sem custo de egresso — o pipeline completo fica dentro do ecossistema GCP.

📄 Docs 💰 Preços
🔷AzureBlob Storage
Vantagens
  • Tiers: Hot, Cool, Cold e Archive
  • Integração nativa com Azure Data Factory e Synapse
  • Azure Data Lake Storage Gen2 para analytics em escala
  • Lifecycle Management automático entre tiers
Desvantagens
  • Terminologia diferente pode confundir (containers ≠ blobs)
  • Performance pode ser inconsistente em regiões menores
💡 Exemplo de uso

Um banco usa Azure Blob Storage como Data Lake com hierarquia de pastas por date partition, processando os dados com Synapse Analytics e movendo arquivos frios para Archive automaticamente após 90 dias.

🗃️
Banco Relacional Gerenciado
PostgreSQL, MySQL, SQL Server gerenciados com backups, patches e HA automáticos
RDS / Aurora Cloud SQL Azure SQL
🟠AWSRDS / Aurora
Vantagens
  • Aurora PostgreSQL/MySQL: até 5x mais rápido que RDS padrão
  • Aurora Serverless v2: escala por frações de ACU
  • Multi-AZ e read replicas automáticos
  • Global Database para replicação multi-região
Desvantagens
  • Aurora pode ser caro para cargas pequenas
  • Vendor lock-in no motor Aurora
💡 Exemplo de uso

Uma SaaS com padrão de uso irregular usa Aurora Serverless v2: durante o dia sobe para 64 ACUs, à noite cai para 0.5 ACU, pagando apenas o que usa — custo mensal 60% menor que uma instância fixa.

📄 RDS 📄 Aurora
🔵GCPCloud SQL / Spanner
Vantagens
  • Cloud SQL: PostgreSQL e MySQL totalmente gerenciados
  • Cloud Spanner: banco relacional distribuído globalmente com SLA 99.999%
  • PITR (Point-in-Time Recovery) nativo
Desvantagens
  • Cloud SQL tem limitações de escalabilidade vs Aurora
  • Spanner é muito caro para cargas pequenas
💡 Exemplo de uso

Um sistema bancário global usa Cloud Spanner para registros de transações que precisam ser consistentes em múltiplos continentes simultaneamente — ACID distribuído com latência de escrita < 10ms entre regiões.

📄 Cloud SQL 📄 Spanner
🔷AzureAzure SQL
Vantagens
  • Melhor SQL Server gerenciado do mercado
  • Intelligent Query Processing automático
  • Hyperscale: escala até 100TB horizontalmente
  • Hybrid Benefit para licenças SQL Server existentes
Desvantagens
  • PostgreSQL/MySQL gerenciados menos maduros que AWS
  • Preço do tier Business Critical é elevado
💡 Exemplo de uso

Uma empresa com ERP legado em SQL Server on-premises migra para Azure SQL Managed Instance com compatibilidade de 100% com SQL Server — zero mudanças no código da aplicação, com HA e backups automáticos.

📄 Azure SQL

DynamoDB Firestore / Bigtable Cosmos DB
🟠AWSDynamoDB
Vantagens
  • Latência single-digit milliseconds garantida
  • On-Demand: paga por request, sem capacity planning
  • DynamoDB Streams para event-driven architectures
  • Global Tables para replicação multi-região
Desvantagens
  • Modelagem de dados complexa (design de chaves)
  • Queries flexíveis limitadas sem índices secundários
  • Custo alto com dados quentes em alta escala
💡 Exemplo de uso

Uma plataforma de jogos armazena sessões de 50 milhões de usuários simultâneos no DynamoDB On-Demand — leituras em < 5ms, escala automática em picos de eventos sem configuração e custo zero em tempo ocioso.

📄 DynamoDB
🔵GCPFirestore / Bigtable
Vantagens
  • Firestore: sincronização em tempo real para apps mobile
  • Bigtable: banco de wide-column para petabytes (usado pelo Google internamente)
  • Firestore tem modo serverless sem capacidade provisionada
Desvantagens
  • Bigtable tem custo mínimo alto (mínimo 1 nó)
  • Firestore tem limitações em queries ad-hoc
💡 Exemplo de uso

Um app de delivery usa Firestore para sincronizar a localização do entregador em tempo real — a posição GPS atualiza a cada 3 segundos no banco e o cliente vê no mapa instantaneamente via Firebase SDK.

📄 Firestore 📄 Bigtable
🔷AzureCosmos DB
Vantagens
  • Multi-API: MongoDB, Cassandra, Gremlin, Table na mesma plataforma
  • Distribuição global com um clique para qualquer região Azure
  • SLA de 99.999% de disponibilidade
  • 5 níveis de consistência configuráveis
Desvantagens
  • Custo pode ser alto — RU/s (Request Units) confusos
  • Compatibilidade MongoDB não é 100% — algumas features faltam
💡 Exemplo de uso

Um marketplace global migra seu MongoDB on-premises para Cosmos DB API MongoDB — zero mudanças no código, dados replicados automaticamente para 5 regiões (Europa, Américas, Ásia) com latência < 15ms localmente.

📄 Cosmos DB
Rede & CDN VPC, DNS, entrega de conteúdo
Redes virtuais privadas, balanceamento, DNS gerenciado e distribuição de conteúdo global.
🌐
CDN — Content Delivery Network
Distribuição de conteúdo estático e dinâmico em pontos de presença globais
CloudFront Cloud CDN Azure CDN / Front Door
🟠AWSCloudFront
Vantagens
  • 600+ PoPs (pontos de presença) globais
  • Lambda@Edge para lógica na borda
  • Integrado nativamente com S3, ALB e API Gateway
  • WAF integrado com regras gerenciadas
Desvantagens
  • Preço de egresso pode ser alto em alto volume
  • Invalidação de cache tem custo adicional
💡 Exemplo de uso

Um portal de notícias distribui imagens e vídeos pelo CloudFront — usuários em São Paulo, Paris e Tóquio recebem o conteúdo do PoP mais próximo em < 20ms, com Lambda@Edge adicionando headers de segurança em tempo real.

📄 CloudFront
🔵GCPCloud CDN
Vantagens
  • Usa a rede global privada do Google (não a internet pública)
  • Cache automático com HTTP cache headers
  • Integrado ao Cloud Load Balancing
Desvantagens
  • Menos PoPs que AWS CloudFront
  • Menos recursos de edge computing
💡 Exemplo de uso

Um app de mapas usa Cloud CDN para cachear tiles de mapa estáticos — 95% das requisições são servidas do cache, reduzindo carga no backend e aproveitando a rede privada do Google para latências consistentes.

📄 Cloud CDN
🔷AzureFront Door
Vantagens
  • Azure Front Door: CDN + LB + WAF numa solução só
  • Roteamento baseado em latência, prioridade ou geolocalização
  • Health probes e failover automático entre backends
Desvantagens
  • Custo pode ser alto para tráfego pequeno
  • Configuração mais complexa que CloudFront
💡 Exemplo de uso

Uma empresa com apps em 3 regiões Azure usa Front Door para rotear usuários para a região mais rápida — se a região primária cai, o Front Door detecta em < 30s e redireciona para o backup automaticamente.

📄 Front Door

Uma startup usa GKE Autopilot para seus 30 microserviços — sem equipe de infra dedicada, o Google gerencia 100% dos nós, patches, escalabilidade e HA. A equipe foca só no código, não na infraestrutura.

Functions as a Service (FaaS)
Execução de funções event-driven sem provisionamento de infraestrutura
Lambda Cloud Functions Azure Functions
🟠AWSLambda
Vantagens
  • 1 milhão de execuções/mês grátis para sempre
  • Cold start mínimo com Provisioned Concurrency
  • Suporte a containers Docker até 10GB
  • Triggers de 200+ serviços AWS nativamente
Desvantagens
  • Timeout máximo de 15 minutos
  • Cold start pode ser lento em runtimes Java/.NET
  • Debugging local mais complexo

Um sistema de processamento de imagens usa Lambda — cada upload no S3 dispara automaticamente uma função que redimensiona em 4 tamanhos, extrai metadados EXIF e salva no DynamoDB, tudo em < 2 segundos por imagem.

📄 Lambda
🔵GCPCloud Functions / Run
Vantagens
  • Cloud Run: containers serverless com concorrência por instância
  • Sem cold start para requests simultâneos (Cloud Run)
  • Cloud Functions Gen2 suporta até 60 minutos de timeout
  • Tráfego zero = custo zero (escala a zero automático)
Desvantagens
  • Cloud Functions tem menos triggers que Lambda
  • Ecossistema de ferramentas serverless menor
💡 Exemplo de uso

Uma API REST dockerizada é deployada no Cloud Run — durante o dia serve 50k req/min escalando para 200 instâncias, à noite escala a zero e o custo é R$0, diferente de uma VM que custaria 24h/dia.

📄 Cloud Functions 📄 Cloud Run

  • Suporte nativo a C#, F# e PowerShell

Desvantagens
  • Cold start em Consumption plan pode ser lento
  • Plano Premium (sem cold start) é caro
💡 Exemplo de uso

Um workflow de aprovação de documentos usa Durable Functions — o processo orquestra: upload → OCR → revisão humana → aprovação → notificação, mantendo estado entre etapas que podem levar dias, sem um servidor rodando.

IAM — Gerenciamento de Identidade

AWS IAM Cloud IAM Azure AD / Entra ID
🟠AWSIAM + Cognito
Vantagens
  • Políticas JSON granulares por recurso e ação
  • IAM Roles para serviços (sem credentials hardcoded)
  • AWS Organizations para governança multi-conta
  • Cognito para autenticação de usuários finais

📄 IAM 📄 Cognito
🔵GCPCloud IAM

  • Service Accounts com Workload Identity Federation
  • Bindings mais simples que políticas JSON da AWS
  • Context-Aware Access para acesso baseado em contexto
Desvantagens
  • Roles predefinidos podem ser muito amplos
  • Custom Roles têm limite de 300 permissões
💡 Exemplo de uso

Uma empresa configura Workload Identity Federation no GCP — pipelines do GitHub Actions acessam Cloud Storage sem nenhuma chave de API armazenada, usando tokens OIDC do GitHub diretamente como credenciais temporárias no GCP.

📄 Cloud IAM
🔷AzureEntra ID (Azure AD)

Desvantagens

💡 Exemplo de uso

Uma empresa com 5.000 funcionários usa Azure AD com Conditional Access — acesso ao ERP SAP exige MFA + dispositivo gerenciado + localização corporativa. Alguém tentando logar de país diferente é bloqueado automaticamente.
